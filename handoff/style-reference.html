how it runs (Framer Motion)
  1. 0.0–0.4s — Cloudinary upload begins. Polaroid stays in undeveloped gradient state. Caption text crossfades to developing….
  2. 0.4–0.9s — Photo <img> fades in over the gradient (opacity 0→1). The img has CSS filter: contrast(1.4) brightness(1.2) saturate(0.7) animating to filter: none — the "chemicals settling" look.
  3. 0.7–1.0s — Grain overlay flashes opacity 0.07 → 0.18 → 0.07.
  4. 0.9–1.4s — Polaroid scale settles: 0.98 → 1.02 → 1.0 (spring stiffness 220, damping 18). Rotation eases by 0.5°.
  5. 1.2–1.6s — Caption swaps from developing… to actual mono date stamp (apr · '26) with a 200ms slide-up.
Trigger: when real_image_url changes from null to a value in Firestore. Use Framer's useAnimationControls() + sequenced controls.start(). Don't use confetti or sparkles — the showpiece is the photo emerging.
Add sheetBottom sheet over a blurred wall. Polaroid border preserved exactly — 8px white frame, ~26px bottom margin for the date stamp, soft paper shadow, gradient placeholder for undeveloped. New when · zone chip row picks which timeline zone the polaroid lands in.
